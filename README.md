# متجر قصّة — دليل الإعداد

## الملفات
- `index.html` — الهيكل فقط
- `style.css` — كل التنسيقات
- `script.js` — كل منطق التطبيق (المتجر + لوحة الإدارة + الصلاحيات)
- `supabase-setup.sql` — يجب تشغيله **مرة واحدة** في مشروع Supabase الخاص بك
- ضعي الملفات الثلاثة الأولى في نفس المجلد بالضبط (نفس الأسماء) عند الرفع على GitHub Pages أو أي استضافة أخرى

## خطوة الإعداد (مرة واحدة فقط)
1. افتحي مشروعك على supabase.com
2. من القائمة الجانبية: **SQL Editor** → **New query**
3. الصقي محتوى ملف `supabase-setup.sql` كاملاً واضغطي **Run**
4. هذا سينشئ:
   - جدول `settings` لرقم الواتساب (متزامن سحابيًا لكل من يفتح الموقع)
   - جدول `admins` لحسابات الدخول، **محمي بالكامل** بحيث لا يستطيع أحد قراءته مباشرة من المتصفح
   - دوال قاعدة بيانات (RPC) تتحقق من الصلاحية داخل السحابة نفسها قبل تنفيذ أي عملية حساسة

## بيانات الدخول الافتراضية
- اسم المستخدم: `admin`
- كلمة المرور: نفس كلمة المرور القديمة التي كانت في ملفك الأصلي (لم تتغير)
- بعد أول دخول، يفضّل تغييرها من تبويب **الإعدادات**

## كيف تعمل الصلاحيات
| العملية | مديرة المتجر (owner) | مشرفة جديدة (staff) |
|---|---|---|
| مراجعة الحجوزات | ✅ | ✅ |
| إضافة/حذف منتجات | ✅ | ❌ (لا تظهر التبويبة أصلاً) |
| تغيير كلمة المرور | ✅ | ❌ |
| تغيير رقم واتساب | ✅ | ❌ |
| إضافة مشرفة جديدة | ✅ (من تبويب "المشرفات") | ❌ |

المشرفة الجديدة تُضاف دائمًا بدور **staff** تلقائيًا، فلا يمكن لأي شخص إنشاء حساب "مديرة متجر" ثانٍ من الواجهة.

## ملاحظة أمنية مهمة (صريحة، بدون مبالغة)
كل بيانات المنتجات والحجوزات في مشروعك الأصلي كانت أصلاً مفتوحة الكتابة عبر مفتاح anon العام (أي مفتاح ظاهر في كود الصفحة لأي زائر). أبقيت هذا كما هو ليعمل الموقع بنفس الطريقة.
لكن بيانات الدخول (اليوزر وكلمة المرور) الآن **محمية داخل قاعدة البيانات نفسها**: لا يمكن لأي زائر — حتى لو فتح "أدوات المطور" في المتصفح وحاول استدعاء الجدول مباشرة — أن يقرأ كلمات المرور أو يضيف نفسه كمديرة متجر، لأن كل عملية حساسة تمر عبر دالة تتحقق من كلمة مرور مديرة المتجر أولاً.

القيد الوحيد المتبقي: منع "المشرفة" من حذف منتج يعتمد حاليًا على إخفاء الزر من الواجهة فقط، وليس على منع حقيقي داخل قاعدة البيانات (لأن ذلك يتطلب نظام حسابات كامل عبر Supabase Auth بدل نظام اليوزر/الرمز المبسّط الحالي). إذا أردت مستقبلاً منع ذلك بشكل كامل حتى من "الكونسول"، أخبريني وسأبني نظام صلاحيات أعمق باستخدام Supabase Auth.
